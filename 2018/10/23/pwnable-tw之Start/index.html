前言

接触CTF的比赛多了之后，越觉得自古Misc、Web留不住，唯有二进制得人心！
所以奋不顾身地投入了二进制的学习之中。在经历CTF从入门到放弃之后，觉得还是需要把学习过程放到博客上。这次也是写了个入门题的Writeup，希望以后能实时更新，不要偷懒！！！

Start

拿到题目先来个三板斧，先objdump -x看看文件格式，然后gdb <filenmae>接一手checksec观察文件保护机制。最后放入IDA，一道美味的pwn题就可以放弃了。不得不说上来第一题还是很友好的，保护机制形同虚设。

关于这些保护的详细描述网络上已经很多了，我们不多copy，直接开始解题。


先gdb运行一下看看程序的流程，用b _start下断点开始调试
发现在int 0x80需要输入，搜索了一下知道int 0x80是linux上的系统调用。
解题就是要莽！先写入20个a看看结果

发现它覆盖了ret的内容，那么思路就很明确了，覆盖返回地址为我们想要的区域，然后写入shellcode
shellcode可以放到栈中，也就是esp中，但esp的地址又是变化的，这怎么办呢？
关键操作就是mov ecx, esp把esp的地址传给了ecx
我们第一次先利用该操作获得esp，第二次再把shellcode写入。
通过gdb易知mov ecx, esp这行地址为0x8048087，而我们输入的起始地址0xffffd224到返回地址0xffffd238刚好是20个字节。Payload1如下：
payload1 = 'a'*20 + p32(0x8048087)
此时获得了esp地址，继续构造第二次的Payload即可，有一点需要注意的是
add esp, 14h，这行操作使得esp的地址加上了0x14。
payload2 = 'a'*20 + p32(20+addr) + shellcode
